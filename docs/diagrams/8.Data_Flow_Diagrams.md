# Data Flow Diagrams (DFD)

Data Flow Diagrams illustrate how data is processed by the KajTrack system in terms of inputs and outputs. They visualize where data comes from, where it goes, and how it gets stored.

## 1. Level 0 DFD (Context Diagram)

The Context Diagram shows the KajTrack system as a single high-level process and illustrates its interactions with external entities (Admin and Employee).

```mermaid
flowchart TD
    %% External Entities
    Admin[Admin]
    Emp[Employee]
    
    %% High-level System
    System((0. KajTrack Task\nManagement System))
    
    %% Data Flows
    Admin -->|"Login Credentials<br>User Info (Create/Edit)<br>Task Details (Create/Assign)"| System
    System -->|"Dashboard Stats<br>User Management List<br>Task Tracking Info<br>Notifications"| Admin
    
    Emp -->|"Login Credentials<br>Task Updates/Status<br>Profile Updates"| System
    System -->|"Assigned Tasks<br>Notifications<br>Profile Data"| Emp
```

## 2. Level 1 DFD (General System Flow)

The Level 1 DFD breaks down the main system into major sub-processes (Authentication, User Management, Task Management, Notifications) and introduces the Data Stores.

```mermaid
flowchart TD
    %% External Entities
    Admin[Admin]
    Emp[Employee]
    
    %% Processes
    Auth((1. Authentication))
    UserMgmt((2. User Management))
    TaskMgmt((3. Task Management))
    NotifMgmt((4. Notification System))
    
    %% Data Stores
    DB_Users[(D1: Users DB)]
    DB_Tasks[(D2: Tasks DB)]
    DB_Notifs[(D3: Notifications DB)]
    
    %% Authentication Flow
    Admin & Emp -->|"Credentials"| Auth
    Auth <-->|"Validate Credentials"| DB_Users
    Auth -->|"Auth Status & Session"| Admin & Emp
    
    %% User Management Flow
    Admin -->|"Add/Edit/Delete User Data"| UserMgmt
    Emp -->|"Update Own Profile"| UserMgmt
    UserMgmt <-->|"Read/Write Users"| DB_Users
    UserMgmt -->|"User List / Feedback"| Admin & Emp
    
    %% Task Management Flow
    Admin -->|"Create/Assign/Edit Task"| TaskMgmt
    Emp -->|"Update Task Progress"| TaskMgmt
    TaskMgmt <-->|"Read/Write Tasks"| DB_Tasks
    TaskMgmt -->|"Task Lists & Statuses"| Admin & Emp
    
    %% Notification Flow
    TaskMgmt -->|"Task Events (Assign/Complete)"| NotifMgmt
    NotifMgmt <-->|"Store/Fetch Alerts"| DB_Notifs
    NotifMgmt -->|"Unread Alerts"| Admin & Emp
```

## 3. Level 2 DFD (Task Management Details)

This Level 2 DFD zooms into the **Task Management** process to show the specific data flow between task creation, assignment, progression, and querying.

```mermaid
flowchart TD
    %% External Entities
    Admin[Admin]
    Emp[Employee]
    
    %% Sub-processes
    P3_1((3.1 Create Task))
    P3_2((3.2 Assign/Edit Task))
    P3_3((3.3 Update Task Status))
    P3_4((3.4 Query Tasks))
    
    %% Data Stores
    DB_Tasks[(D2: Tasks DB)]
    DB_Users[(D1: Users DB)]
    
    %% Task Creation
    Admin -->|"New Task Input"| P3_1
    P3_1 -->|"Insert Task"| DB_Tasks
    
    %% Task Assignment
    Admin -->|"Assignment/Edits"| P3_2
    P3_2 <-->|"Verify Employee Existence"| DB_Users
    P3_2 -->|"Update Assignee"| DB_Tasks
    
    %% Task Progression
    Emp -->|"Change Status (Pending/WIP/Done)"| P3_3
    P3_3 -->|"Update Status Flag"| DB_Tasks
    
    %% Query Tasks
    Admin & Emp -->|"View My Tasks / All Tasks"| P3_4
    P3_4 <-->|"Fetch Task Records"| DB_Tasks
    P3_4 -->|"Display Records"| Admin & Emp
```
