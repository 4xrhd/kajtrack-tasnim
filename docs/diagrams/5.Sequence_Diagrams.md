# Sequence Diagrams

Sequence diagrams represent the timeline of messages exchanged between the user, the frontend views, the backend handlers, and the database.

## 1. User Login Sequence
```mermaid
sequenceDiagram
    participant User
    participant Browser
    participant AuthHandler
    participant Database

    User->>Browser: Enters credentials & clicks Login
    Browser->>AuthHandler: POST /handlers/login.php
    AuthHandler->>Database: SELECT * FROM users WHERE username = ?
    Database-->>AuthHandler: Returns user record
    
    alt Invalid Credentials
        AuthHandler-->>Browser: Redirect (Location: login.php?error=...)
        Browser-->>User: Displays error message
    else Valid Credentials
        AuthHandler->>AuthHandler: Verify Password
        AuthHandler->>AuthHandler: session_start() & Set Session Vars
        AuthHandler-->>Browser: Redirect (Location: index.php)
        Browser-->>User: Loads Dashboard
    end
```

## 2. Admin Creates Task Sequence
```mermaid
sequenceDiagram
    participant Admin
    participant Browser
    participant TaskHandler
    participant Database

    Admin->>Browser: Submits new task form
    Browser->>TaskHandler: POST /handlers/create_task.php
    
    TaskHandler->>TaskHandler: Validate inputs (title, desc, employee_id)
    
    alt Validation Fails
        TaskHandler-->>Browser: Redirect to form with error
    else Validation Succeeds
        TaskHandler->>Database: INSERT INTO tasks (title, description, assigned_to...)
        Database-->>TaskHandler: Confirm insertion
        
        TaskHandler->>Database: INSERT INTO notifications (message, recipient...)
        Database-->>TaskHandler: Confirm notification creation
        
        TaskHandler-->>Browser: Redirect to tasks.php with success
        Browser-->>Admin: Displays task list with success alert
    end
```
