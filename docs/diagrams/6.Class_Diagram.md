# Class Diagram / Domain Model

Since KajTrack is primarily built using a procedural PHP approach with handlers and views, this "Class Diagram" represents the Domain Entities and Logical Modules rather than strict Object-Oriented PHP classes. 

```mermaid
classDiagram
    class User {
        +int id
        +string full_name
        +string username
        +string password
        +string role
        +login()
        +logout()
        +updateProfile()
    }

    class Task {
        +int id
        +string title
        +string description
        +int assigned_to
        +date due_date
        +string status
        +createTask()
        +updateStatus()
        +deleteTask()
    }

    class Notification {
        +int id
        +string message
        +int recipient
        +string type
        +date date
        +boolean is_read
        +createNotification()
        +fetchUnreadCount()
        +markAsRead()
    }

    class DatabaseConnection {
        -string sName
        -string uName
        -string pass
        -string db_name
        -PDO conn
        +connect()
    }

    class Handlers {
        <<System Controllers>>
        +login_handler()
        +task_handler()
        +user_handler()
        +notification_handler()
    }

    User "1" -- "*" Task : Manages / Assigned
    User "1" -- "*" Notification : Receives
    Handlers --> DatabaseConnection : Uses
    Handlers ..> User : Controls
    Handlers ..> Task : Controls
    Handlers ..> Notification : Controls
```
